## PHASE 0 — SERVER-SIDE RENDERING FIX (P0 CRITICAL BLOCKER)

**Test:** Run this mental model — if a search engine crawler or an Apple App Store reviewer visits construktr.ai, do they see HTML content or a blank page that requires JavaScript to render?

If the site is client-side rendered only (empty HTML shell, all content loaded via JS), this is a **hard blocker** for:
- Google indexing (Google will see an empty page)
- App Store review (Apple reviewers clicking privacy/terms URLs may see blank pages → REJECTION)

**Fix:** Convert the site to serve pre-rendered HTML. Options in order of preference:
1. **Static Site Generation (SSG)** — generate complete HTML at build time (preferred for a 5-page marketing site)
2. **Server-Side Rendering (SSR)** — render HTML on first request
3. **Pre-rendering middleware** — serve static HTML snapshots to crawlers

**Verification test after fix:**
```bash
curl -s https://construktr.ai | grep -c "<h1\|<h2\|<p\|<section"
```
Must return > 0. If it returns 0, the fix did not work.
